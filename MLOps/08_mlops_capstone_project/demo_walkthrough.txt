PART 1 — CODE WALKTHROUGH  (~2 min)
============================================================

--- Show: capstone_flow.py ---

NARRATION:
"This is our main Metaflow pipeline. It defines a single linear DAG with 9 steps."

ACTION: Scroll through the class, point out each step decorator:
  1. start         — initializes MLflow (tracking URI, experiment, registered model)
  2. load_data     — loads reference and batch parquets via capstone_lib
  3. integrity_gate — two-layer gate:
     • Layer 1: hard rules (missing columns, invalid datetimes,
       negative durations, range violations) — "reject_batch" if any fail
     • Layer 2: NannyML soft checks (missingness drift via
       UnivariateDriftCalculator, unseen categoricals) — sets integrity_warn tag
  4. feature_engineering — stable schema (FEATURE_COLS: 16 columns), credit-card filter, datetime features, duration,
        clipping, log transforms, frequency encoding, interaction features. Logs feature_cols.json with dtypes to MLflow.
  5. load_champion — loads models:/<name>@champion; if none exists,
     bootstraps by training on reference and registering with @champion alias
  6. model_gate    — evaluates champion on batch vs reference:
     • computes champion_rmse, baseline_rmse, rmse_increase_pct
     • logs predictions.parquet (champion inference on batch)
     • logs evaluation dataset lineage via mlflow.data
     • decides retrain_needed (>5 % RMSE increase, lowered to 3 % if integrity_warn is true)
     • always writes decision.json
  7. retrain       — if retrain_needed:
     • trains candidate on reference ∪ batch
     • evaluates on batch AND on reference (stability check P3)
     • logs candidate model, predictions.parquet, feature_cols.json
     • logs training dataset lineage
  8. promotion_gate — four criteria (P1–P4):
     P1: evaluation metrics exist + dataset lineage logged
     P2: candidate RMSE < champion × (1 − min_improvement)
     P3: candidate doesn't regress reference by >5 %
     P4: no hard integrity failures
     If all pass → register new version, flip @champion alias, tag old champion as previous_champion.
     Always logs decision.json.
  9. end           — summary log

ACTION: Show the Parameters at the top:
  • reference-path, batch-path (required)
  • tracking-uri (default localhost:5001)
  • experiment-name (default 08_capstone_green_taxi)
  • model-name (default green_taxi_tip_model)
  • min-improvement (default 1 %)

NARRATION:

ACTION: Briefly highlight:

APPENDIX: TROUBLESHOOTING
--------------------------
• If Run 2 does NOT trigger retrain (RMSE increase is below 5%):
  Lower the threshold by appending to the run command:
    --min-improvement 0.20
  This makes the promotion bar higher, but the retrain threshold in model_gate is hardcoded at 5%/3%.
  If the champion happens to perform well on the August batch, you can temporarily edit model_gate to lower the
    threshold (e.g., 0.01 instead of 0.05).

• If the MLflow server is not running:
    mlflow server --workers 1 --port 5001 --backend-store-uri sqlite:///mlflow_tracking/mlflow.db --default-artifact-root mlflow_tracking/mlruns 

• If "No module named 'metaflow'" or similar:
    conda activate 22971-mlflow
    pip install metaflow nannyml
